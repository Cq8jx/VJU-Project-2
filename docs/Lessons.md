## 記録済みの教訓

### [2026-02-20] Import Maps の欠如で起動クラッシュ（設計段階で予防）

**状況:** PRD レビューにて発覚
**エラー内容:** `Uncaught TypeError: Failed to resolve module specifier "react"` が予測された
**原因:** ビルドステップなしで `import { useState } from 'react'` を書いた場合、ブラウザはベアモジュールを解釈できない
**解決策:** `index.html` に `<script type="importmap">` を必ず定義し、esm.sh 経由でライブラリを解決する
**教訓:** Google AI Studio (Browser-only) では **Import Maps を最初に定義することが絶対条件**。後から追加は困難
**他プロジェクトへの応用可能性:** ✅ Node.js を使えない任意のブラウザ実行環境に適用できる

---

### [2026-02-20] GitHub API レート制限（設計段階で予防）

**状況:** PRD レビューにて指摘
**エラー内容:** GitHub REST API は未認証で 60回/時 の制限がある
**原因:** `/repos/.../contents/` API を頻繁に叩く設計だった
**解決策:** API 呼び出しを撤廃し、GitHub Pages で静的ホストした `data.json` を直接 fetch する
**教訓:** **GitHub をDB代わりにする場合は API ではなく静的ファイル（Raw/Pages）を使う**
**他プロジェクトへの応用可能性:** ✅ 静的サイトジェネレーターで事前にJSONをビルドするパターン全般

---

### [2026-02-20] BrowserRouter でリロード時 404（設計段階で予防）

**状況:** PRD レビューにて指摘
**エラー内容:** `/detail/001` などの URL でリロードすると GitHub Pages が 404 を返す
**原因:** pushState ベースのルーティングはサーバー側の Rewrite 設定が必要
**解決策:** **HashRouter** (`#/detail/001`) を使用する。GitHub Pages は `index.html` の 404 フォールバック設定が限定的
**教訓:** サーバー設定が自由にできない静的ホスティングでは必ず HashRouter を選択
**他プロジェクトへの応用可能性:** ✅ S3 静的ホスティング、Netlify（独自設定なし）でも同様
