# Requirements.md — GlobalNotice VN

> **最終更新:** 2026-02-20
> **対応PRD:** GlobalNotice_VN_PRD_v2.md

---

## 1. Goal / Success Criteria（目標と達成基準）

| # | 目標 | 達成基準 |
|---|------|---------|
| G1 | 大学のお知らせ（GlobalNotice）をブラウザで快適に閲覧できる | お知らせ一覧が2秒以内に表示される |
| G2 | オフライン時も直前に閲覧したコンテンツが利用できる | IndexedDB キャッシュにより再接続なしで閲覧可能 |
| G3 | Google AI Studio 環境で動作する | Node.js ビルドなしでブラウザ単体で動作する |
| G4 | PDF 添付ファイルをアプリ内で閲覧できる | モバイル・デスクトップ両方で閲覧またはダウンロードが可能 |

---

## 2. User Stories（ユーザーストーリー）

### 教職員・学生（一般利用者）
- `US-01` お知らせ一覧を日付・カテゴリでフィルタリングして探したい
- `US-02` お知らせの詳細（本文・PDF）をアプリ内で読みたい
- `US-03` オフライン環境でも直前に見たお知らせを再確認したい
- `US-04` スマートフォンからも快適に利用したい

### 管理者（コンテンツ管理）
- `US-05` GitHub リポジトリに JSON ファイルをコミットするだけでアプリへ反映したい
- `US-06` アプリの動作に問題が発生した場合、デバッグログをダウンロードして確認したい

---

## 3. Scope / Non-Scope

### 対象範囲（In Scope）
- お知らせ一覧・詳細表示
- PDF 添付ファイルの閲覧・ダウンロード
- クライアントサイドの検索・フィルタ機能
- IndexedDB によるオフラインキャッシュ
- デバッグログのエクスポート機能

### 対象外（Non-Scope）
- サーバーサイド処理・バックエンド API の新設
- ユーザー認証（ログイン機能）
- お知らせの作成・編集 UI（CMSは GitHub）
- Push 通知
- Node.js / npm ビルドプロセス（**厳禁**）

---

## 4. Non-Functional Requirements（非機能要件）

| カテゴリ | 要件 |
|---------|------|
| **パフォーマンス** | `Promise.all` による並列フェッチ。初回表示 ≤ 2秒（キャッシュあり ≤ 0.5秒） |
| **信頼性** | `ErrorBoundary` による部分障害の隔離。API エラー時はリトライ停止してUI通知 |
| **セキュリティ** | API キー・PAT をクライアントコードに含めることを禁止 |
| **ユーザビリティ** | Toast 通知優先、モーダルは最小限。PDF はフォールバック DL リンクを必須配置 |
| **メモリ** | PDF Object URL は閲覧後に `revokeObjectURL` で解放 |
| **ホスティング** | GitHub Pages（静的）または直接ファイル起動で動作すること |
